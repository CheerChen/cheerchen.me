# Blog Taxonomy

This file is a soft guide for choosing categories and tags when creating or editing posts.
Prefer the existing taxonomy, but add a new tag or category when the article clearly needs it.

## Categories

Keep categories broad and stable. A new category should usually be useful for more than one future post.

### 技术 / 技術

Cloud, architecture, engineering practice, development tools, troubleshooting, infrastructure, and technical conference reports.

Examples:
- AWS Summit session reports
- Docker / Homelab notes
- Git and developer tooling
- Architecture or backend design analysis

### AI 协作 / AI協働

AI-assisted work, LLM collaboration, coding agents, prompt and workflow design, and AI tool reflections.

Examples:
- Claude Code / Codex / Gemini workflows
- AI-assisted debugging
- Personal CLI tools for AI review
- Reflections on AI collaboration patterns

### 职业与学习 / キャリアと学習

Career development, learning methods, personal study notes, and non-core engineering learning.

Examples:
- Career or workplace reflections
- Learning finance or investing with AI
- Study notes that are more about learning process than implementation

## LLM Editing Notes

When generating or editing a post:

1. Read this file before choosing categories and tags.
2. Prefer one existing category.
3. Add a new category only when the article does not naturally fit the existing ones.
4. Tags may be added more freely, but reuse existing tags where possible.
5. For bilingual posts, keep the category and tag meanings aligned across `index.zh.md` and `index.ja.md`.
6. Do not force exact translation when a language has a more natural term, such as `树莓派` and `Raspberry Pi`.
